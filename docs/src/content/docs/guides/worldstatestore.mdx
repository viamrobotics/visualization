This is the workflow for: **engineers writing a Viam `WorldStateStoreService` module**. The service streams obstacles, frames, and other spatial entities to clients (motion planners, debug viewers, etc.) as `*commonpb.Transform` protos. The `draw` package handles the tedious parts — converting a `spatialmath.Geometry` into a `commonpb.Geometry`, deriving stable UUIDs, and packing color/visibility metadata.

If you only need to render snapshots in a browser, see [Embedding `<MotionTools />`](/guides/embedding/) instead — that path doesn't require implementing a service.

## What you're building

A `WorldStateStoreService` exposes three RPCs:

- **`ListUUIDs`** — every transform you currently track.
- **`GetTransform(uuid)`** — full data for one transform.
- **`StreamTransformChanges`** — live `ADDED` / `UPDATED` / `REMOVED` events.

The interesting piece is the stream: clients subscribe once and rely on you to emit deltas. That means your service has to maintain internal state, diff it against new data, and broadcast the diff.

This guide walks through that loop with one concrete example: an **obstacle store** that polls a Viam sensor named `obstacle-sensor`, reconciles the readings against its internal state, and streams the deltas. The sensor's `Readings()` returns a list of obstacles, each with a position and dimensions. The store treats position as the identity — same position, different dimensions ⇒ `UPDATED`; previously-seen position missing from a tick ⇒ `REMOVED`.

<Aside type="note">
	A real implementation would probably use a stable ID provided by the sensor as the identity, not
	position. We're indexing by position here to keep the example self-contained — the sensor isn't
	required to emit IDs, and quantized position works as a deterministic key for synthetic test
	scenarios. Swap in whatever identity makes sense for your data.
</Aside>

## The store

### State and lifecycle

```go
package main

import (
    "context"
    "fmt"
    "math"
    "sync"
    "time"

    "github.com/golang/geo/r3"
    "github.com/viam-labs/motion-tools/draw"
    commonpb "go.viam.com/api/common/v1"
    pb "go.viam.com/api/service/worldstatestore/v1"
    "go.viam.com/rdk/components/sensor"
    "go.viam.com/rdk/logging"
    "go.viam.com/rdk/resource"
    "go.viam.com/rdk/services/worldstatestore"
    "go.viam.com/rdk/spatialmath"
)

const pollInterval = time.Second

type obstacleStore struct {
    resource.Named
    resource.TriviallyReconfigurable
    resource.TriviallyCloseable

    logger logging.Logger
    sensor sensor.Sensor

    mu         sync.RWMutex
    transforms map[string]*commonpb.Transform // keyed by position key

    streamCtx context.Context
    cancel    context.CancelFunc

    subsMu sync.Mutex
    subs   map[chan worldstatestore.TransformChange]struct{}
}

func newObstacleStore(
    ctx context.Context,
    deps resource.Dependencies,
    conf resource.Config,
    logger logging.Logger,
) (worldstatestore.Service, error) {
    obstacleSensor, err := sensor.FromDependencies(deps, "obstacle-sensor")
    if err != nil {
        return nil, fmt.Errorf("getting obstacle-sensor: %w", err)
    }

    streamCtx, cancel := context.WithCancel(context.Background())
    s := &obstacleStore{
        Named:      conf.ResourceName().AsNamed(),
        logger:     logger,
        sensor:     obstacleSensor,
        transforms: make(map[string]*commonpb.Transform),
        streamCtx:  streamCtx,
        cancel:     cancel,
        subs:       make(map[chan worldstatestore.TransformChange]struct{}),
    }

    go s.pollLoop()
    return s, nil
}

func (s *obstacleStore) Close(ctx context.Context) error {
    s.cancel()
    s.subsMu.Lock()
    for ch := range s.subs {
        delete(s.subs, ch)
        close(ch)
    }
    s.subsMu.Unlock()
    return nil
}
```

`subs` is the fan-out for `StreamTransformChanges`. Every active subscription owns one channel; `broadcast` writes to all of them. Concurrent clients don't starve each other.

### The required service methods

```go
func (s *obstacleStore) ListUUIDs(ctx context.Context, extra map[string]any) ([][]byte, error) {
    s.mu.RLock()
    defer s.mu.RUnlock()

    uuids := make([][]byte, 0, len(s.transforms))
    for _, t := range s.transforms {
        uuids = append(uuids, t.Uuid)
    }
    return uuids, nil
}

func (s *obstacleStore) GetTransform(ctx context.Context, uuid []byte, extra map[string]any) (*commonpb.Transform, error) {
    s.mu.RLock()
    defer s.mu.RUnlock()

    for _, t := range s.transforms {
        if string(t.Uuid) == string(uuid) {
            return t, nil
        }
    }
    return nil, resource.NewNotFoundError(resource.NewName(worldstatestore.API, string(uuid)))
}

func (s *obstacleStore) StreamTransformChanges(
    ctx context.Context,
    extra map[string]any,
) (*worldstatestore.TransformChangeStream, error) {
    ch := make(chan worldstatestore.TransformChange, 100)

    s.subsMu.Lock()
    s.subs[ch] = struct{}{}
    s.subsMu.Unlock()

    // Tear down the subscription when the client disconnects.
    go func() {
        <-ctx.Done()
        s.subsMu.Lock()
        if _, ok := s.subs[ch]; ok {
            delete(s.subs, ch)
            close(ch)
        }
        s.subsMu.Unlock()
    }()

    return worldstatestore.NewTransformChangeStreamFromChannel(ctx, ch), nil
}
```

These are mechanical. The interesting work happens in `pollLoop` and `reconcile`.

### Polling and reconciliation

```go
type obstacle struct {
    X, Y, Z    float64
    SX, SY, SZ float64
}

func (s *obstacleStore) pollLoop() {
    ticker := time.NewTicker(pollInterval)
    defer ticker.Stop()

    for {
        select {
        case <-s.streamCtx.Done():
            return
        case <-ticker.C:
            obstacles, err := s.readObstacles()
            if err != nil {
                s.logger.Warnw("failed to read obstacles", "err", err)
                continue
            }
            s.reconcile(obstacles)
        }
    }
}

func (s *obstacleStore) readObstacles() ([]obstacle, error) {
    readings, err := s.sensor.Readings(s.streamCtx, nil)
    if err != nil {
        return nil, err
    }

    raw, ok := readings["obstacles"].([]any)
    if !ok {
        return nil, fmt.Errorf("sensor returned no obstacles slice")
    }

    out := make([]obstacle, 0, len(raw))
    for _, r := range raw {
        m, _ := r.(map[string]any)
        if m == nil {
            continue
        }
        out = append(out, obstacle{
            X: floatField(m, "x"), Y: floatField(m, "y"), Z: floatField(m, "z"),
            SX: floatField(m, "sx"), SY: floatField(m, "sy"), SZ: floatField(m, "sz"),
        })
    }
    return out, nil
}

func floatField(m map[string]any, key string) float64 {
    if v, ok := m[key].(float64); ok {
        return v
    }
    return 0
}
```

`reconcile` is where the diff happens. We build the next set of position keys, then walk both maps to figure out adds, updates, and removals:

```go
func (s *obstacleStore) reconcile(obstacles []obstacle) {
    next := make(map[string]obstacle, len(obstacles))
    for _, o := range obstacles {
        next[posKey(o)] = o
    }

    s.mu.Lock()
    defer s.mu.Unlock()

    // ADDED + UPDATED
    for key, o := range next {
        existing, seen := s.transforms[key]
        transform, err := buildTransform(key, o)
        if err != nil {
            s.logger.Warnw("failed to build transform", "err", err, "key", key)
            continue
        }
        s.transforms[key] = transform

        switch {
        case !seen:
            s.emit(transform, pb.TransformChangeType_TRANSFORM_CHANGE_TYPE_ADDED, nil)
        case dimensionsChanged(existing, o):
            s.emit(transform, pb.TransformChangeType_TRANSFORM_CHANGE_TYPE_UPDATED, []string{"physicalObject"})
        }
    }

    // REMOVED — anything in the previous tick that's missing now.
    for key, t := range s.transforms {
        if _, present := next[key]; present {
            continue
        }
        delete(s.transforms, key)
        s.emit(&commonpb.Transform{Uuid: t.Uuid}, pb.TransformChangeType_TRANSFORM_CHANGE_TYPE_REMOVED, nil)
    }
}

// posKey quantizes the position to 1mm so floating-point jitter doesn't
// flip identity. Use whatever resolution matches your sensor.
func posKey(o obstacle) string {
    return fmt.Sprintf("%d:%d:%d",
        int(math.Round(o.X)),
        int(math.Round(o.Y)),
        int(math.Round(o.Z)),
    )
}

func dimensionsChanged(t *commonpb.Transform, o obstacle) bool {
    box := t.GetPhysicalObject().GetBox()
    if box == nil || box.DimsMm == nil {
        return true
    }
    return box.DimsMm.X != o.SX || box.DimsMm.Y != o.SY || box.DimsMm.Z != o.SZ
}
```

### Building a `Transform` with `draw`

This is where `draw` earns its keep. `WithID(key)` makes the UUID deterministic from the position key, so the same obstacle on every poll produces the same UUID — a requirement for `UPDATED` events to match on the client.

```go
func buildTransform(key string, o obstacle) (*commonpb.Transform, error) {
    box, err := spatialmath.NewBox(
        spatialmath.NewZeroPose(),
        r3.Vector{X: o.SX, Y: o.SY, Z: o.SZ},
        key,
    )
    if err != nil {
        return nil, err
    }

    drawn, err := draw.NewDrawnGeometry(box, draw.WithGeometryColor(draw.ColorFromName("orange")))
    if err != nil {
        return nil, err
    }

    return drawn.Draw(key,
        draw.WithID(key),
        draw.WithPose(spatialmath.NewPoseFromPoint(r3.Vector{X: o.X, Y: o.Y, Z: o.Z})),
    )
}
```

For other primitive shapes (sphere, capsule, mesh), substitute the corresponding `spatialmath.New*` constructor — `draw.NewDrawnGeometry` takes any `spatialmath.Geometry`. For point clouds, see [`draw.NewDrawnPointCloud`](/api/draw/#NewDrawnPointCloud).

### Emitting changes

```go
func (s *obstacleStore) emit(
    transform *commonpb.Transform,
    changeType pb.TransformChangeType,
    updatedFields []string,
) {
    change := worldstatestore.TransformChange{
        ChangeType:    changeType,
        Transform:     transform,
        UpdatedFields: updatedFields,
    }

    s.subsMu.Lock()
    subs := make([]chan worldstatestore.TransformChange, 0, len(s.subs))
    for ch := range s.subs {
        subs = append(subs, ch)
    }
    s.subsMu.Unlock()

    for _, ch := range subs {
        select {
        case ch <- change:
        case <-s.streamCtx.Done():
            return
        default:
            // Subscriber is full; drop this change rather than blocking the
            // poll loop. In practice, give them a deeper buffer or a slow-
            // consumer policy that suits your latency requirements.
        }
    }
}
```

## Module bootstrap

```go
// main.go
package main

import (
    "context"

    "go.viam.com/rdk/logging"
    "go.viam.com/rdk/module"
    "go.viam.com/rdk/resource"
    "go.viam.com/rdk/services/worldstatestore"
)

var Model = resource.NewModel("yourns", "obstacles", "store")

func init() {
    resource.RegisterService(
        worldstatestore.API,
        Model,
        resource.Registration[worldstatestore.Service, resource.NoNativeConfig]{
            Constructor: newObstacleStore,
        },
    )
}

func main() {
    ctx := context.Background()
    logger := logging.NewDebugLogger("obstacle-store")

    mod, err := module.NewModuleFromArgs(ctx)
    if err != nil {
        logger.Fatal(err)
    }
    if err := mod.AddModelFromRegistry(ctx, worldstatestore.API, Model); err != nil {
        logger.Fatal(err)
    }
    if err := mod.Start(ctx); err != nil {
        logger.Fatal(err)
    }
    defer mod.Close(ctx)
    <-ctx.Done()
}
```

Configure the module in your machine config with `obstacle-sensor` as a dependency, point a viewer at the part, and you should see boxes appear, resize, and disappear as the sensor reports change.

## What to take from this

The pattern generalizes:

<Steps>

1. Maintain an internal map keyed by the identity you care about (UUID from the source, hashed position, etc.).
2. On each tick (or push event), build the next set, diff against current, and emit `ADDED` / `UPDATED` / `REMOVED`.
3. Use `draw.NewDrawn*` + `Draw(name, draw.WithID(key))` to produce `*commonpb.Transform`s with deterministic UUIDs so updates land on the same entity client-side.
4. Fan out broadcasts so multiple subscribers don't block each other.

</Steps>

For the `draw` API surface — colors, metadata, point clouds — see the [`draw` reference](/api/draw/). Note the world state store service _only_ works with `Transform` objects from the Viam API. It is not able to render `Drawings`.
